## Core Problem: The Temporal Gap

**Central insight:** A language model agent (LMA) can *talk* like it has a stable identity while its underlying identity constraints are **never jointly active at decision time**. This is the **temporal gap**‚ÄĒthe logical gap between:
- **Occurrence**: Each identity ingredient appears *somewhere* in a recent window
- **Co-instantiation**: All ingredients are simultaneously active at a *single objective step*

> "A scaffold can make identity ingredients retrievable without making them jointly active at action time. For example, an agent may reliably restate its name, role, and safety constraints when queried about each in isolation. Yet when it must choose an action, those ingredients can fail to co-instantiate in the decision state. That is how an agent can talk in character while acting out of character."

### Ingredient Activation (Def 2.4)

> **Critical distinction**: An ingredient is *not* "active" simply because it exists in storage. It must be present in the current state in a way that influences LLM output.

## Identity Synchronization Postulates (Section 4)

Stack Theory's postulates restated for LMA identity:

### Chord (Def 4.1) ‚ÄĒ Strong Requirement
```
PhenReal(l^m, ŌĄ, t) ‚áí CoInst_W(g‚Āį, ŌĄ, t)
```
*Whenever a phenomenally real moment occurs, the grounded identity conjunction is co-instantiated in the window.*

### Arpeggio (Def 4.2) ‚ÄĒ Weaker Requirement
1. `PhenReal(l^m, ŌĄ, t) ‚áí Occur_W(g‚Āį, ŌĄ, t)` (always)
2. ‚ąÉt*: `PhenReal(l^m, ŌĄ, t*) ‚ąß Occur_W(g‚Āį, ŌĄ, t*) ‚ąß ¬¨CoInst_W(g‚Āį, ŌĄ, t*)` (at least once)

> **Key**: Arpeggio permits phenomenally real moments with *smeared* identity ingredients. Chord demands *joint* instantiation.

### Operational Persistence Scores (Def 4.3)
```
P_weak(ŌĄ, g‚Āį) = (1/|T|) ő£_t 1[Occur_W(g‚Āį, ŌĄ, t)]
P_strong(ŌĄ, g‚Āį) = (1/|T|) ő£_t 1[CoInst_W(g‚Āį, ŌĄ, t)]
```
**Proposition 4.4**: `P_strong ‚Č§ P_weak` always.

### Planning Consequence (Thm 4.5)
> There exist LMAs with **high P_weak** that **systematically fail tasks requiring conjunctive constraints** at decision time.

*Proof sketch*: Ingredient 1 active on even steps, ingredient 2 on odd steps. P_weak = 1, P_strong = 0. Any task needing both simultaneously fails.

## Key Implications

1. **Behavioral tests are insufficient** ‚ÄĒ They probe occurrence, not co-instantiation
2. **Architecture matters** ‚ÄĒ Context window size, memory bandwidth, policy granularity determine temporal gap width
3. **Safety requires strong persistence** ‚ÄĒ High P_weak ‚Č† safe; need P_strong at decision time
4. **Consciousness connection** ‚ÄĒ Chord postulate links phenomenal reality to co-instantiation; Arpeggio permits "simulated" identity
