OMNIGENT — Resonant Command Center

Research Orchestration Engine • Single-Page Application Architecture

Application Layer

  • • Single-page app: 1 HTML + 1 TSX entry point (~1,154 LOC)
  • • Vite dev server + ES module loading (no bundler in dev)
  • • Three-panel layout: Substrates | Monitor/Lattice | Engine
  • • Dual view modes: Monitor (chat) and Lattice (D3 graph)
  • • Command palette (Ctrl+K) for keyboard-driven navigation
  • • Custom non-blocking modal system (replaces confirm/prompt/alert)

Data & State

  • • localStorage persistence: workspaces + research queue
  • • Runtime state: chat history, graph nodes/links, engine config
  • • D3 force-directed graph: 12 seed concepts, user-injected nodes
  • • Document ingestion pipeline: upload → summarize → index
  • • Research queue: add goals, run pipeline, benchmark τ
  • • safeJsonParse wrapper prevents malformed localStorage crashes

Security & API

  • • Whitelist HTML sanitizer (src/security.ts) on all AI output
  • • URL protocol filtering: blocks javascript:/data:/vbscript:
  • • Event handler stripping on dynamically rendered HTML
  • • API keys stored in-memory only, never in localStorage
  • • OpenRouter gateway: fetch-based, Bearer token auth
  • • Global registry (Map-based) replaces (window as any) pollution
